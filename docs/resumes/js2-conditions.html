La console

La console est très importante. Nous allons la découvrir au fur et à mesure car la pratique et les exemples marchent mieux que des mots. Pour faire court, il s'agit d'un endroit dans le navigateur où l'on peut écrire et exécuter du code JavaScript, ainsi qu'interagir avec les programmes qui tournent.

Pour ouvrir la console, il faut accéder aux outils de développement. Généralement, on appuie sur la touche F12 du clavier sur une page web (sur Windows). Pour Mac, on utilise ⌥⌘C

Les booléens

Un booléen est une variable ou un litéral (une valeur) pouvant posséder deux valeurs:

  • true (vrai)
  • false (faux)

Et c'est tout.

Opérations avec les booléens

Les booléens possèdent trois opérations qui leurs sont propres. Ces opérations prennent un ou deux booléens et en renvoient un troisième. Le résultat de l'opération est décrit dans les paragraphes suivants, par un tableau. La première colonne contient les valeurs de bool1 et la première ligne contient les valeurs de bool2. Le reste du tableau contient le résultat en fonction des valeurs des deux booléens.

ET ou AND

Renvoi true si et seulement si les deux booléens valent true.


bool1 && bool2;

OU ou OR

Renvoi true si au moins un des deux booléens vaut true.


bool1 || bool2;

NON ou NOT

Cette opération n'utilise qu'un seul booléen. La valeur de ce booléen est inversée.


!bool2;

Les opérations qui renvoient un booléen

Certaines opérations entre deux valeurs de types quelconques renvoient des booléens qui peuvent être stockés dans des variables ou utilisée dans des opérations et conditions. Ce sont des opérations de comparaison.

Dans la liste suivante, a et b sont deux variables. Si ces variables ne sont pas de même type, une conversion sera effectuée. Attention donc car ce comportement est parfois imprévisible.

  • a < b: renvoi true si a est strictement inférieur à b.
  • a <= b: renvoi true si a est inférieur ou égal à b.
  • a > b: renvoi true si a est strictement supérieur à b.
  • a >= b: renvoi true si a est supérieur ou égal à b.
  • a == b: renvoi true si a est égal en valeur à b, après conversion. Attention, il y a bien deux signes =. Un seul signifie l'affectation. C'est une erreur inattention courante.
  • a != b: renvoi true si a est différent en valeur de b après conversion.
  • a === b: renvoi true si a est égal à b, en valeur et type. Même si les deux valeurs sont égales (si a == b est vrai), si une conversion a lieue, le résultat sera false.
  • a !== b: renvoi true si a est différent en valeur ou type à b. Même si les valeurs sont égales (a != b est faux), si une conversion a eu lieue, le résultat sera true.

Si la comparaison est vraie, alors le résultat sera true. Sinon, le résultat sera false.

Les conditions

Les conditions sont ce qu'on appelle des structures de contrôle. Les structures de contrôle contrôle la manière dont le code est exécuter et nous permettent d'aller bien plus loin que la simple exécution d'instructions.

Les conditions vont nous permettre d'exécuter du code, seulement si certaines conditions sont remplies. Il existe trois types de structures de conditions.

if ... else if ... else

Ce type de structure utilise des opérations de comparaison et des opérations entre booléens.

if

If est la base de cette structure. Il contient la première condition à évaluer et peut exister seul.


if (condition) {
    //Code
}

Le code entre les accolades {} sera exécuté si la condition est évaluée à true. Si la condition vaut false, le code entre les accolades ne sera pas exécuté et le code continuera après. Après l'exécution du code, le programme reprend son cours normal.

Un code placé entre accolades est appelé bloc. Toutes les instructions seront exécutées en tant qu'une seule.

else

Else permet d'ajouter du code à exécuter dans le cas ou la condition du if est fausse. Il est facultatif. Après exécution, le programme reprend son cours normal.


if (condition) {
    //Code à exécuter si la condition est vraie
} else {
    //Code à exécuter si la condition est fausse
}
else if

Else if se place entre le if et le else. Il peut y en avoir autant que nécessaire. Il est facultatif. La présence d'un else n'est pas non plus obligatoire pour insérer des else if.


if (condition) {
    //Code
} else if (condition) {
    //Code
} else if (condition) {

} ...
} else {
    //Code
}

Le code d'un else if sera exécuté si aucun des conditions du if et des else if précédents n'est vraie (donc si aucun code de la condition n'a été exécuté). Une fois le code exécuté, le programme sort de la condition et reprend son cours normal. Un seul else if n'est donc exécutable.

switch

Switch est une autre forme de condition qui vérifie une égalité.


switch (variable) {
    case val:
        //Code
        break;
    case val2:
        //Code
        break;
    ...
    default:
        //Code
}

On va tester l'égalité de la variable avec les valeurs. Dès que l'on trouve une valeur égale à la variable, le code contenu dans le case est exécuté.

break; permet d'arrêter l'exécution du code et de sortir du switch. Sans cette instruction, le programme continuerai jusqu'au prochain break ou jusqu'à la fin du switch, en passant par plusieurs case. C'est parfois ce que l'on souhaite, mais il faut comprendre cette mécanique.

Enfin, default contient le code à exécuter si aucun case n'a été validé.

Un petit mot sur le code

Le code contenu dans ces structures de contrôle peut également contenir des conditions, mais aussi tout type de structure de controle ou d'instructions. On peut donc imbriquer les if par exemple.

Dernière structure de condition: les conditions ternaires

Ce concept est plus compliqué. Il s'utilise lors de l'affectation et permet de donner une valeur différente à la variable en fonction d'une condition.


variable = (condition)? val1 : val2;

Si la condition est vraie, on renvoi val1 (ici, on affecte val1 à variable). Si la condition est fausse, on renvoi val2.

On peut également imbriquer ce type de conditions.
