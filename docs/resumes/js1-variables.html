HTML

HTML est un langage de description balisé. Il permet de décrire le contenu d'une page web, sans se soucier de la mise en forme.

Ecrire le code HTML

Le code HTML s'écrit dans un fichier d'extension .html. On appelle souvent le fichier principal index.html.

Balises et attributs HTML

On utilise des balises pour décrire le contenu de la page web. Chaque balise possède un nom et représente un certain élément (titre, paragraphe, image, ...). Elles possède un sens (sémantique) et souvent une mise en page par défaut.

Il ne faut pas utiliser une balise pour changer l'apparence du texte. C'est la fonction du langage CSS que nous verrons plus tard. On utilise une balise pour insérer un élément particulier.

Certaines balises peuvent contenir d'autres éléments (balises ou texte). Dans ce cas, elles commencent par une balise ouvrante et se terminent par une balise fermante:


<nomDeLaBalise> <!--Balise ouvrante-->
  	Contenu
</NomDeLaBalise><!--Balise fermante-->

Une balise ne peut être fermée que si les balises qu'elle contient sont fermées également.

Le texte entre <!-- et --> est ce qu'on appelle un commentaire. Il s'agit d'une indication laissée par le programmeur pour les personnes lisant le programme. Ces informations ne seront pas lues par le naviateur. Attention, puisque tout le monde à accès à ce texte, il ne faut pas y mettre d'informations sensibles (comme des mots de passe).

On peut compléter une balise avec des attribut. Les attributs permettent de donner des détails sur l'utilisation de la balise, de fournir des informations au navigateur ou de modifier le contenu de la balise:


<nomDeLaBalise nomDeLAttribut="valeur">
    Contenu
</nomDeLaBalise>

Un élément peut posséder plusieurs attributs, il suffit de les mettre les uns à la suite des autres.

Certains attributs ne possèdent pas de valeur.

Certaines balises ne peuvent rien contenir. Elles permettent parfois d'insérer un élément ou de fournir des données au navigateur. On les utilise souvent avec des attributs:


<nomDeLaBalise attributOptionnel="valeur" autreAttributSansValeur />

Page de base en HTML: "Hello, World!"

La page "Hello, World!" est souvent la première page créée (et de manière générale le premier programme créé). C'est un programme simple qui affiche une sorte de bonjour. Mais le cette page est également un excellent moyen d'observer la structure de base d'un programme. Toute les pages écrites devront au minimum avoir la structure suivante (sauf le contenu de <body> et de <title> qui vont changer en fonction de la page.


<!DOCTYPE html> <!--Cette ligne indique au navigteur que la page contient du code HTML.-->

<html><!--La balise &lt;html&gt; englobe tout le contenu-->
    <head> <!--Cette balise contient des métadonnées, informations utiles au navigateur mais généralement non visibles par l'utilisateur.-->
        <meta charset="utf-8" /><!--Permet de choisir le set de caractères. utf-8 permet d&#39;utiliser presque tous les caractères (alphabet latin, idéogrammes asiatiques, arabe, emojis, signes mathématiques, ...).-->
        <title>Titre</title><!--Titre de la page, parfois visible sur l&#39;onglet.-->
    </head><!--Fermeture de la balise <head>-->

    <body><!--La balise <body> contient tout le contenu visible de la page.-->
        <!--Dans cette page, nous avons un titre <h1> ainsi qu'un paragraphe <p>-->
        <h1>Hello, World!</h1>
        <p>Bonjour tout le monde!</p>
    </body><!--Fermeture de la balise <body>-->

</html><!--Fermeture de la balise <html>-->

JavaScript

Ecrire le code JavaScript

Nous pouvons écrire le code JavaScript de deux manières:

  • Soit dans un fichier indépendant d'extension .js. Cette méthode est à favoriser car elle permet une maintenance et une réutilisation du code plus simple. On nomme souvent le fichier principal script.js.

<script src="script.js"></script>
  • Soit directement dans la balise <script>. Cette méthode est à éviter.

<script>
  	Code JavaScript
</script>

Chaque instruction JavaScript se termine par un ;. Rappelons d'une instruction est une étape de l'algorithme.

Il est également possible d'insérer des commentaires en JavaScript, comme pour HTML. Il existe deux types de commentaires:

  • Les commentaires de ligne qui commencent par // et se terminent à la fin de la ligne:

instruction; //On commente cette instruction
  • Les blocs de commentaire qui commencent par /* et se terminent par */; et peuvent occuper plusieurs lignes:

instruction; /*Début du commentaire
On continue le commentaire
Fin du commentaire*/

Variables en JavaScript

Une variables est un nom à qui on peut donner une valeur.Les variables permettent de garder en mémoire une valeur pour la réutiliser plus tard ou la modifier.Les variables possèdent un cycle de vie. en JavaScript:

  • La déclaration et l'initialisation. La déclaration permet de créer une variable et l'initialisation lui donne sa première valeur. L'initialisation est facultative.

let nomDeVariable; //Déclaration seule
let nomDeVariable = valeur; //Déclaration et initialisation

Il est important de donner un nom clair à une variable, sans abbréviations. Si on souhaite avoir un nom avec plusieurs mots (comme dans l'exemple), on colles les mots et chaque mot commence par une majuscule.

  • L'utilisation et l'affectation. Utiliser une variable revient à récupérer sa valeur. L'affectation consiste à donner une nouvelle valeur à la variable. 

console.log(nomDeVariable);//utilisation de la variable pour l'afficher
nomDeVariable = nouvelleValeur; //Affectation
nomDeVariable = nomDeVariable + 1;//Utilisation et affectation

le dernier exemple utilise la même variable des deux côtés du =. Cette instruction ajoute 1 à la valeur de la variable. la navigateur va d'abord utiliser la variable dans l'addition, puis utiliser le résultat pour l'affectation.

Valeurs des variables

Une variable peut posséder différents types de valeurs:

Les nombres (Number)

On peut utiliser des nombres entiers ou décimaux.

Contrairement au français, en programmation nous utilisons le point . et non la virgule , en guise de séparateur décimal

Voici quelques exemples:

  • 1
  • 0
  • 586
  • -6558
  • 0.22584
  • -7854.5
Les chaînes de caractères (String)

Ce sont des suites de 0 caractères ou plus.

On les place entre guillemets, simples ou doubles.

Voici quelques exemples:

  • "Je suis une chaîne de caractères"
  • '4'
  • "" (Il s'agit de la chaîne vide)
  • '...'
Les booléens (Boolean)

Nous en apprendrons un peu plus sur les cours suivants. Pour faire simple, c'est un type qui possède deux valeurs:

  • true (vrai)
  • false (faux)
Undefined

Unedefined est le type d'une variable qui n'a pas été initialisée.

Object

Comme pour les booléens, nous en apprendrons plus dans les cours suivant.

Une valeur de ce type nous intéresse cependant: null. Cette valeur représente l'absence de valeur et est très utile. Nous verrons des exemples régulièrement pendant les cours.

Opérations

Voici la liste des opérations de bases en JavaScript:

  • a + b est l'addition quand les deux membres sont des nombres. Si l'un d'entre eux est un chaîne de caractères, alors il s'agit de l'opérateur de concaténation. La concaténation permet de mettre bout à bout des chaînes de caractères pour en faire une plus grande.
  • a - b est la soustraction.
  • a * b est la multiplication. Attention, il s'agit de la touche étoile * et non de la touche x.
  • a / b est la division.
  • a % b est le modulo. Le modulo est une opération peu connue. Il s'agit de prendre le reste de la division euclidienne (ou division entière) de a par b.
  • a = b est l'affectation, comme nous l'avons déjà vu.

Comme en mathématiques, les opérations ont un ordre de priorité: la multiplication et la division sont prioritaires sur l'addition et la soustraction. Cet ordre de priorité peut être modifié avec l'utilisation de parenthèses ().
