Le guide du débutant

Introduction

La programmation est une activité accessible à tous, mais on peut vite être découragé si l’on n’a pas l’habitude. Comment fonctionne mon logiciel ? Comment taper ce caractère ?...

Ce document est là pour répondre à certaines questions que l’on se pose quand on débute et ainsi vous aider à vous concentrer sur le principal : créer un programme et s’amuser.

Ce cours se concentre sur la création de pages web et de scripts JavaScript, à exécuter dans le navigateur. La plupart des conseils restent cependant valable pour tout langage de programmation.

De quel matériel ai-je besion pour commencer la programmation?

L’avantage de la programmation web est qu’elle ne nécessite pas de matériel précis. Il vous faudra tout de même un ordinateur (Windows, Mac ou Linux, peu importe) avec une connexion internet et un navigateur web récent (Google Chrome, Mozilla Firefox, Safari, Microsoft Edge, … mais surtout pas Internet Explorer, sauf si on aime s’arracher les cheveux).

Il vous faudra également un éditeur de texte pour écrire le code. Attention, les logiciels comme Microsoft Word ou Libre Office, qui permettent de créer des documents et de les mettre en page, ne fonctionneront pas.

Les éditeurs de texte

Que choisir ?

Je vous propose d’utiliser l’un des suivants :

  • Visual Studio Code : créé par Microsoft, il est très populaire (et c’est mon favoris). Je l’utiliserai généralement pendant le cours.
  • Notepad++ : le plus simple de tous. Il n’est disponible que sous Windows.
  • Sublime Text : un éditeur de texte puissant, qui ressemble beaucoup à Visual Studio Code. Attention, ce logiciel peu vous demander une licence pour l’utiliser. Mais vous n’êtes pas obligé. Si jamais vous ne pouvez plus l’utiliser, choisissez-en un autre. Ne payez jamais pour un éditeur de texte.
  • Brackets : encore un éditeur puissant créé par Adobe.

Attention, certains logiciels sont plus difficiles à prendre en main. A moins que vous ayez l’habitude, je vous recommande de ne pas utiliser ni Atom, ni Vim, ni Emacs.

Les éditeurs de texte facilitent l’écriture de code grâce à plusieurs procédés :

  • Coloration syntaxique. Les différents mots clefs, noms de variables et de fonctions ou valeurs seront colorés différemment pour faciliter la lecture.
  • Autocomplétion. Une liste de mots utilisables peut apparaître lors de l’écriture du code. Si le mot désiré est dans cette liste, on peut le choisir et appuyer sur Entrée pour l’insérer.
  • Gestion de plusieurs fichiers. On peut facilement travailler sur plusieurs fichiers à la fois. On retrouve généralement nos documents ouverts dans des onglets au dessus de la zone d’écriture.
  • Certains éditeurs plus puissants proposent d’autres options comme la fermeture automatique des balises HTML.

Comment écrire mon code ?

Quand on n’a pas l’habitude d’écrire au clavier, ça peut parfois prendre du temps. Et ce n’est pas grave. Plus on écrit, plus on va vite.

Dans les premiers cours, vous n’aurez qu’à recopier ce que j’écris à l’écran, nous prendrons notre temps et attendrons tout le monde.

Remarques et conseils

  • La programmation est universelle. Du coup, la plupart des fonctions et outils sont en anglais. C’est perturbant au début mais on fini par s’y faire. En plus de progresser en programmation, on progresse en anglais !
  • Pendant les cours je vous parlerai d’indentation. Il s’agit de placer une ou plusieurs tabulations avant la ligne de code lorsque l’on entre dans un bloc. Dans certains langages, c’est obligatoire. Avec JavaScript, ce n’est pas obligatoire mais conseillé pour la lisibilité.
  • Attention à l’orthographe ! Si une faute de frappe est présente, le programme pourrait ne pas fonctionner.
  • Pour faciliter la programmation, il est possible sur PC et Mac de placer différentes fenêtres côte à côte sur l’écran. Pour Windows, il suffit de déplacer la fenêtre sur le bord gauche ou droit, et pour Mac, cliquer et glisser la fenêtre vers le bord en maintenant le bouton vert enfoncé.
  • Pour finir, le code écrit ne sera visible que par les personnes ayant accès à votre ordinateur, à moins de le mettre en ligne volontairement.

Introduction à la programmation web

Algorithmique

Un algorithme est une suite d’instructions permettant de résoudre un problème ou d’obtenir un résultat. Ils ne se limitent pas à l’informatique :

  • Une recette de cuisine est un algorithme.
  • Une tactique en sport est un algorithme.
  • Panser un cheval est un algorithme.
  • Résoudre un casse-tête nécessite souvent l’emploi d’algorithmes.
  • …

Le web

Le web est une connexion entre plusieurs machines du monde entier. Il ne faut pas le confondre avec internet, qui est un ensemble de service dont le web fait parti.

Comment fonctionne le web ?

Le client (vous) va demander une page web (par exemple YouTube). Cette demande va être envoyée à un serveur (un gros ordinateur très puissant) qui va générer la page et vous la renvoyer.

Une page web est en réalité du code informatique. Il existe trois langages pour créer les pages web, chacun avec son utilité. Ce sont ces langages que nous étudieront dans cette activité.

HTML

HTML est le langage permettant de créer la structure de la page web et de donner son contenu (texte, images, formulaires, …). On ne l’utilise pas pour faire la mise en page mais pour donner du sens à la page.

Si on le compare à une maison, il s’agirait des fondations et des briques.

CSS

CSS est le langage permettant de donner du style à la page. On peu changer les couleurs du texte, ajouter des bordures aux éléments, changer leur position…

Si on le compare à une maison, ce serait les décorations, la peinture ou la tapisserie.

JavaScript

JavaScript est le langage permettant de rendre la page plus dynamique. On pourra interagir avec l’utilisateur, créer des effets, animations, et même des jeux ou des applications.

Ce langage sera le point central de cette activité. Nous l’utiliserons avec la bibliothèque P5.js. Une bibliothèque est un ensemble de fonctionnalités déjà codées. L’utiliser permet de ne pas avoir à recoder ces fonctionnalités et ainsi à se concentrer sur l’application. Ne pas l’utiliser, c’est un peu comme si il fallait récolter et moudre son blé pour faire la farine à chaque fois que l’on fait un gâteau. Pas très pratique…
